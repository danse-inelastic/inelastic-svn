Known Bugs

User Feedback

  • Chunking does not work.

  • Table Support is partially implemented.

  • Lists column management , half - fixed.

  • When the user specifies a preamble or overrides some templates, there may exist undefined commands (e.g. \hypertarget without the hyperref package). Check.

  • Extra spaces generated at the end of segtitle.

  • \usepackage{fancybox} prevents toc and lof, even added by hand, to be generate: we get empty shells (try it directly in the TeX).

  • biblioset is not supported (Workaround, need to work on this).

  • biblioentry have funny final punctuation (dangling commas and the like).

  • character not supported in the text Check out all iso-latin-1 while you're at it.

  • Feature requests: . I would like the thing to handle languages in documents (article lang="fr". Right now I add by hand the needed.

  • \usepackage for input encoding and active characters.

  • Aesthetics remarks: sometimes you use {} and sometimes not in the preamble (\usepackage).

  • itemizedlists inside tables will cause LaTeX errors unless column widths are specified (but tabularx is fine).

  • Ordered lists don't support 'continues' (only 'enumerate').

  • longtable/widetable

  • Trouble with screen/verbatim environments inside tables, admonition.

  • Alternative presentations for revhistory

  • Empty hypertarget problem (paragraph breaks) with latex (as opposed to pdftex).

  • Using a style file for makeindex can cause incorrect index generation if it involves changing the “quote” character. This may conflict with the makeindex -g option.

Further Investigation Required

  • Footnotes: if possible, improve upon the existing \catcode kludge for \urls within \footnotes. Do index links work inside footnotes? How can footnotes work in tables?

  • Page-spanning floats in two-column reports. Add a Q&A.

  • Handling of character entities. Make a user guide about this.

  • authorgroup and copyright ranges -- localise rather than use hard-coded "x, y, and z".

  • Provide a more compact \maketitle for articles.

  • Support per-chapter ToC. Does LaTeX already provide a facility?

  • Support for regular typefaces in literallayout.

  • Support for subtitles.

  • Support for tocbibind.

  • LaTeX whitespace distance after acronyms/abbrevs that terminate with mid-sentence full stops.

  • Parameterise remaining \sc usage.

  • itemizedlist in varlistentry?

  • Better hooks, in places where it can be provided (e.g. figures).

  • Verbatim in tables

  • \refs inside \hyperlinks

  • Processing of PIs

  • Persistent links for reference documentation.

  • Different modes for long headers (e.g. two-per-page or chap on one page and section on the other.)

Long-term Projects

  • Documentation.

  • Index support.

  • xrefs to unusual types such as anchors.

  • Use xmllint --valid on samples.

  • Proper tables support.

  • bibliomixed support.

  • Componentinfo support. Support of abstracts in more places?

  • Improved verbatim-type support.

  • Q&A support (including hyperlinks).

  • Support multiple section-level indices, etc. (Already have biblio support.)

  • MathML support.

  • Add chunking, so each chapter/part/etc... is created on a separate .tex file and manage the \input. (Also: automation of cover file generation.) May support $rootid attribute, though LaTeX hyperlinking won't work.
